# ADR-002: 使用 Subprocess 隔离替代 Docker 沙箱执行 Trace 捕获

## 状态：已接受
## 上下文：
Sprint 2 的任务要求实现 Harness (基于推测包裹技术的执行引擎)，拦截任意不受信的代码或脚本执行时的返回态（包括死代码，内存外溢死锁和异常退出）。最初设想在 Docker 内做沙箱。

## 决策：
使用纯 Python 原生的 `asyncio.create_subprocess_shell`。放弃守护进程型的强隔离方案。通过捕捉底层 POSIX 文件指针（stdout/stderr PIPE）直接重温结果，并强加强制性 Timeout 保证不会产生僵尸进程（Zombie）。

## 理由：
1. 本阶段主要是证明 SECA 对于单机脚本的“推理诊断回路”能走通。过早引入 Docker Daemon 要求繁杂，影响迭代速度。
2. 通过 asyncio 并发管理执行管道可以直接配合我们在 Sprint 1 决定的 `aiosqlite` 非阻塞基座。

## 后果：
此方案不防范恶意用户执行 `rm -rf /`，只能保证普通系统调用的回馈（防溢性弱）。待整系统成型后的演进期可替换 `sandbox.py` 下的 `execute_command` 实现，对上层 Orchestrator 的 API 无感知。
