# ADR-004: 选用 Mermaid.js 替代 D3 进行 DAG 回放树渲染

## 状态：已接受
## 上下文：
在 Sprint 4 的设计需求中，需要在内省仪面板增加包含历史纠错节点的回退链路可视化（Playbook Tree）。最开始考虑采用灵活底层的图计算库 D3.js。

## 决策：
直接引入并使用基于纯文本声明式语法的 `mermaid.js`（尤其是它的 `graph TD` 支持）。同时由前端组装生成合法的 Mermaid syntax 给予框架直接加载。

## 理由：
1. D3.js 具有陡峭的入门曲线，并会极大提升 React 组件内的代码复杂度及其对 DOM 操作侵入的反模式问题。
2. Mermaid 通过简洁的连线语法（如 `1 --> 2`）非常自然且极其对应地贴合了我们在 SQLite 存储的 `parent_trace_id` -> `id` 关系树层级。降低了维护负担。

## 后果：
由于 Mermaid 在处理海量级（>500 节点）图表时性能渲染会出现延迟甚至堵塞浏览器主线程，我们需要注意回溯图不能无限制堆积。可以在未来增加限制拉取数。
