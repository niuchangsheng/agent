# ADR-001: SQLite 环境与 SQLAlchemy 异步驱动的使用

## 状态：已接受
## 上下文：
在 Sprint 1 中，我们需要为 SECA 搭建本地知识与状态的持久化基础库。作为初始 MVP 版本，避免复杂的 Docker Compose 网络配置，选择本地 SQLite 数据库作为存储最优解。由于后续需要接入 SSE (Server-Sent Events) 与前端双向通信机制以及复杂的异步 Agent 并行任务，数据库调用绝对不能阻塞 FastAPI 主线程的 Event Loop。

## 决策：
在 FastAPI 栈中使用 `SQLModel` 作为元数据的声明层，通过注入 `aiosqlite` 配置为标准的非阻塞异步数据库连接层 (`create_async_engine`) 。

## 理由：
1. `SQLModel` 完美结合了 SQLAlchemy 声明式优势和 Pydantic 的验证与 Type Hints；能通过同一套模型复用到 API 返回验证，高度保持一致并降低代码冗余度。
2. 异步 SQLite 允许开发无缝从本地过渡至后期的云架构 `PostgreSQL`（只需将驱动切换至 `asyncpg` 即可完美平移，无代码逻辑修改）。

## 后果：
测试环节带来了一定配置成本（如需单独撰写使用 `engine.begin().run_sync()` 的代码强制进行事件环中创建表实体操作）。但在生产环境执行任务的吞吐将获得显著的非阻塞提升。
