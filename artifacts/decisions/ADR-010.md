# ADR-010: API Key 加密存储升级 bcrypt

**状态**: 已采纳 (2026-04-13)

## 上下文

Sprint 8 实现了 API Key 的 SHA256 哈希存储。Sprint 10 升级为更安全的 bcrypt 算法，原因：

1. **SHA256 的局限**:
   - 计算速度快，容易被暴力破解
   - 无内置盐值机制（虽然可以添加，但非标准用法）
   - 专为完整性校验设计，非密码存储

2. **bcrypt 的优势**:
   - 内置随机盐值，相同密码产生不同哈希
   - 可调节计算成本（rounds 参数）
   - 专为密码存储设计，抗暴力破解

## 决策

采用 **bcrypt** 替代 SHA256 作为 API Key 哈希算法。

### 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 哈希算法 | bcrypt | 专为密码存储设计，抗暴力破解 |
| Rounds | 12 | 平衡安全性和性能（约 250ms/次） |
| 库 | `bcrypt` (PyPI) | 官方维护，C 实现性能好 |

### 实现变更

```python
# auth.py 升级
def hash_api_key(key: str) -> str:
    salt = bcrypt.gensalt(rounds=12)
    return bcrypt.hashpw(key.encode(), salt).decode()

def verify_api_key(key: str, key_hash: str) -> bool:
    return bcrypt.checkpw(key.encode(), key_hash.encode())
```

### 验证逻辑变更

由于 bcrypt 每次产生不同哈希（随机盐），验证逻辑需要调整：

```python
# 旧实现（SHA256）：直接比较哈希值
SELECT * FROM api_key WHERE key_hash = hash(input_key)

# 新实现（bcrypt）：遍历验证
for key in all_keys:
    if verify_api_key(input_key, key.key_hash):
        return key
```

## 权衡

### 优势 (+)
- 安全性大幅提升，抗暴力破解
- 随机盐值，同 Key 不同哈希
- 符合 OWASP 密码存储最佳实践

### 劣势 (-)
- 验证性能下降（需遍历所有 Key）
- 哈希长度增加（60 字符 vs 64 字符）
- 现有 API Key 需重新创建（不兼容）

### 向后兼容
- **不迁移旧 Key**: 旧 Key 使用 SHA256，新 Key 使用 bcrypt
- **验证逻辑兼容**: 同时支持两种算法（可选）
- **当前实现**: 全部升级为 bcrypt，旧 Key 需重新创建

## 影响

### 技术影响
- `auth.py` 完全重写哈希和验证逻辑
- 新增 `verify_api_key()` 函数
- 测试用例新增 bcrypt 验证测试

### 运维影响
- 现有用户需重新创建 API Key
- 性能影响：验证时间从 ~1ms 增至 ~250ms（可接受）

### 安全影响
- ✅ 符合安全合规要求
- ✅ 抵抗彩虹表攻击
- ✅ 抵抗暴力破解
