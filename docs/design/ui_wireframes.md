# SECA UI 线框图与页面流转

## 用户旅程总览

```text
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           SECA 用户旅程闭环                                       │
│                                                                                 │
│   [0. 发令台] ──提交──> [1. 内省总览仪] ──完成──> [2. 回溯长廊] ──导出──> [ADR]  │
│       ↑                    ↑                      ↑                    ↑        │
│    用户输入              执行监控                 审查回放             知识沉淀    │
│                                                                                 │
```

## 1. 整体基调与审美准则
- **色阶**: 黑客感冷深系，基础背景以碳灰/靛蓝色(`bg-slate-950`)为主。采用 Glassmorphism 毛玻璃特性（背挂浅荧光渐变晕染）以体现高度专业透明化的前卫工具感。
- **指示色**: 通过极光青 (`text-cyan-400`) 对比当前追踪事件；血红 (`border-red-500` 呼吸特效) 表示被 Harness 沙箱制止的灾难阻断情况。

### 0.3 组件规格

| 元素 | 类型 | 验证规则 | 样式 |

### 4.2 用户操作路径

| 起点 | 操作 | 终点 | 触发条件 |
|------|------|------|----------|
| 发令台 | 提交任务 | 任务队列 | 表单验证通过 + API 成功 |
| 任务队列 | 点击运行任务 | 内省仪 | 任务状态 = RUNNING |
| 任务队列 | 点击已完成任务 | 回溯廊 | 任务状态 = COMPLETED |
| 内省仪 | 任务完成 | 回溯廊 | 自动跳转 (可选) |
| 回溯廊 | 返回上级 | 任务队列 | 点击返回按钮 |
| 任意页面 | 提交新任务 | 发令台 | 点击顶部"提交任务"按钮 |

## 6. 设计反思：为什么 v1.0-v1.3 漏掉了发令台？

| 问题层级 | 具体表现 | 本次修正 |
|---------|---------|---------|
| **Wireframe 设计** | 只画了执行中 + 完成后，跳过了启动前 | 新增第 0 章：任务发令台 |
| **用户旅程分析** | 假设"任务已启动"，没有设计入口 | 新增用户旅程总览图 + 状态机 |
| **导航设计** | 只有监控类 Tab，没有发起类按钮 | 新增顶部"提交任务"按钮 |
| **页面流转** | 没有定义如何从一个页面到另一个 | 新增第 4 章：页面流转与状态机 |
| **闭环验证** | 没有验证用户能否完成完整旅程 | 新增第 5.2 章：闭环验证清单 |

**教训**: 设计文档必须从"用户打开页面"开始，走到"用户获得价值"结束。任何断点都是 P0 缺陷。
