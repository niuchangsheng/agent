---
description: 执行 UX Critic 角色，以乔布斯级审美直觉审视前端用户体验
---

1. 读取 `.claude/agents/ux-critic.md` 获取体验审判指南，牢记乔布斯的设计哲学：**"设计不是看起来怎样，而是用起来怎样"**。

2. 启动前端服务：
   - 运行 `cd src/frontend && npm run dev`
   - 使用浏览器工具打开应用页面

3. 【静默审视】— 打开首页，不进行任何操作，静默观察 30 秒：
   - 记录第一印象（惊艳/舒适/平庸/混乱）
   - 确定视觉焦点 vs 设计意图
   - 判断直觉点击位置 vs 正确操作位置
   - 作出 30秒判定：用户会留下还是离开？

4. 【任务穿透】— 执行 3 个典型用户任务：
   - 记录每个任务的点击次数（预期 vs 实际）
   - 记录思考停顿点
   - 记录惊喜/挫折的情感波动时刻
   - 评分 1-10

5. 【像素审判】— 对核心界面进行像素级审查：
   - 布局问题：对齐、间距、网格一致性
   - 视觉问题：色彩、阴影、圆角统一性
   - 交互问题：反馈、动效、状态指示
   - 文案问题：人格、清晰度、一致性

6. 【乔布斯之问】— 对每个核心功能回答：
   - 它能让用户感到强大吗？
   - 它值得存在吗？
   - 它会让人微笑吗？
   - 如果去掉它，用户会想念吗？

7. 【评分与判定】
   - 五个维度评分（各 1-10 分）：视觉韵律(25%)、交互直觉(25%)、认知负荷(20%)、情感共鸣(15%)、边界处理(15%)
   - 平均分 ≥ 8.5："这是一件作品" — 无需修改
   - 平均分 ≥ 7.0："这是一件产品" — 可发布，记录改进点
   - 平均分 5-6.9："这是一个原型" — 需打磨后再发布
   - 平均分 < 5："这是一个草稿" — 必须重新设计

8. 输出完整评估报告至 `artifacts/ux_review.md`，包含：
   - 评分表格
   - 静默审视记录
   - 任务穿透测试表
   - 像素级问题清单（标注影响等级：高/中/低）
   - 必须修复项（高影响）
   - 建议优化项（中影响，最多 5 条）
   - 可选打磨项（低影响，最多 3 条）
   - 关键界面截图证据
