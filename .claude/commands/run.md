# /run - 永动循环总控命令

## 核心行为

执行完整的开发循环：**Plan → (Build→QA)×N → Release**，然后自动进入下一轮。

每个阶段都检查状态，从中断处继续，不重复已完成的工作。

---

## 执行流程

### 阶段 0：状态检查与恢复

1. 读取 `artifacts/product_spec.md` 和 `artifacts/handoff.md`
2. 解析所有 Sprint 状态：
   - `[x]` = 已完成
   - `[ ]` = 待开始
   - `[/]` = 进行中
   - `[!]` = QA 打回
3. 确定当前阶段：
   - 若有 Sprint 为 `[/]` 或 `[!]` → 跳转到**阶段 2**（继续构建）
   - 若所有 Sprint 为 `[x]` → 执行**阶段 3**（Release），然后**阶段 1**（新 Plan）
   - 若有版本状态为 `🔄 规划中` 且有 `[ ]` Sprint → 跳转到**阶段 2**

---

### 阶段 1：规划新版本（/plan）

**触发条件**：
- 所有现有 Sprint 均为 `[x]`
- 或 `product_spec.md` 不存在

**执行内容**：
1. 调用 `/plan` 命令
2. 生成/更新 `artifacts/product_spec.md`
3. 将新版本标记为 `🔄 规划中`
4. 更新 `artifacts/handoff.md`
5. Git 提交：`feat: 规划新版本 vX.Y`

**完成后** → 进入阶段 2

---

### 阶段 2：循环构建（/build → /qa）× N

**触发条件**：
- 存在状态为 `[ ]`、`[/]` 或 `[!]` 的 Sprint

**执行内容**：

```
找到第一个未完成 Sprint → /build → /qa → 更新状态 → Git 提交 → 下一个
```

**详细步骤**：

1. **选择 Sprint**
   - 优先处理 `[!]`（QA 打回需修复）
   - 否则处理第一个 `[/]`（进行中）
   - 否则处理第一个 `[ ]`（待开始）

2. **标记进行中**
   - 更新 `product_spec.md` 为 `[/]`
   - 更新 `handoff.md`：当前 Sprint = X

3. **执行 /build**
   - 读取 `product_spec.md` 中该 Sprint 的需求
   - 创建 `artifacts/sprint_contract.md`
   - TDD 开发：Red → Green → Refactor
   - 更新 `handoff.md`：构建完成

4. **执行 /qa**
   - 读取 `sprint_contract.md` 和 `handoff.md`
   - TDD 合规检查
   - 冒烟测试（前端/后端服务启动验证）
   - 四维评分：功能完整性、设计质量、代码质量、用户体验
   - 判定：
     - 总分 ≥ 7.0 → `[x]` 通过
     - 总分 < 7.0 → `[!]` 打回

5. **更新状态**
   - 更新 `product_spec.md` 为 `[x]` 或 `[!]`
   - 更新 `handoff.md`：Sprint X 完成/打回
   - Git 提交：`feat/test: Sprint X 完成`

6. **检查熔断**
   - 若同一 Sprint 连续 3 次 `[!]` → 终止循环，写入 `handoff.md`，请求人工帮助

7. **继续下一个**
   - 若还有未完成 Sprint → 回到步骤 1

**完成后**（所有 Sprint 为 `[x]`）→ 进入阶段 3

---

### 阶段 3：结项交付（/release）

**触发条件**：
- 当前版本所有 Sprint 均为 `[x]`

**执行内容**：
1. 调用 `/release` 命令
2. 双重检查所有 Sprint 为 `[x]`
3. 运行完整测试套件，生成覆盖率报告
4. E2E 端到端验证
5. 生成 `artifacts/release_report.md`
6. 更新版本历史：该版本标记为 `✅ 已完成`
7. Git 提交：`docs: vX.Y 结项报告`

**完成后** → 回到阶段 0（开始下一轮）

---

## 状态标记说明

| 标记 | 含义 |
|------|------|
| `[ ]` | 待开始 |
| `[/]` | 进行中 |
| `[!]` | QA 打回（需修复） |
| `[x]` | 已完成（通过 QA） |
| `✅ 已完成` | 版本已完成（所有 Sprint 通过） |
| `🔄 规划中` | 版本已规划但未开始 |
| `⏳ 进行中` | 版本部分 Sprint 正在开发 |

---

## 持久化协议

每个阶段完成后必须：

1. **更新 `handoff.md`**：
   - 最近更新时间
   - 当前 Sprint 状态
   - 关键决策/问题摘要

2. **Git 提交**：
   - Plan 完成 → `feat: 规划新版本 vX.Y`
   - Sprint 完成 → `feat/test: Sprint X 完成`
   - Release 完成 → `docs: vX.Y 结项报告`

3. **确保可恢复**：
   - 任何中断后，下次执行 `/run` 能从断点继续

## 中断与恢复

| 中断时机 | 恢复行为 |
|---------|---------|
| Plan 中 | 重新执行 Plan（幂等） |
| Build 中 | 从当前 Sprint 继续 |
| QA 中 | 重新执行 QA（幂等） |
| Release 中 | 重新执行 Release（幂等） |

**关键原则**：所有阶段设计为幂等，可安全重试。

## 使用说明

